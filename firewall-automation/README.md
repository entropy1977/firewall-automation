# Firewall Rules Automation

A Python-based tool for managing, auditing, and documenting firewall rules across enterprise environments. Automates rule generation from YAML policies, detects conflicts and redundancies, and produces compliance-ready documentation.

## Why I Built This

Managing firewall rules across multiple devices in a multi-site enterprise is complex and error-prone. In my experience administering Cisco and Fortinet firewalls for eight departments, I found that manual rule management led to redundant entries, conflicting policies, and poor documentation. This tool automates the tedious parts — generating consistent rules from policy definitions, auditing existing rulesets for issues, and producing clear documentation for compliance reviews.

## Features

- **Policy-as-Code** — define firewall policies in human-readable YAML; generate device-specific rules automatically
- **Rule audit** — analyze existing rulesets to find redundant, shadowed, or conflicting rules
- **Compliance reporting** — generate documentation showing which business requirements map to which firewall rules
- **Multi-format output** — produces rules in Cisco ACL and iptables format (extensible to other vendors)
- **Change tracking** — logs all rule changes with timestamps and justification for audit trails

## Configuration

Define your security policies in `policies.yaml`:

```yaml
policies:
  - name: "Allow Web Traffic"
    description: "Permit HTTP/HTTPS from internal network to internet"
    source: "192.168.0.0/16"
    destination: "any"
    ports: [80, 443]
    protocol: "tcp"
    action: "permit"
    zone_from: "INSIDE"
    zone_to: "OUTSIDE"
    business_justification: "Required for staff internet access"

  - name: "Block Telnet"
    description: "Deny all Telnet traffic (insecure protocol)"
    source: "any"
    destination: "any"
    ports: [23]
    protocol: "tcp"
    action: "deny"
    zone_from: "any"
    zone_to: "any"
    business_justification: "Security policy - unencrypted protocol"

  - name: "Allow DNS"
    description: "Permit DNS queries to internal DNS servers"
    source: "192.168.0.0/16"
    destination: "192.168.1.5"
    ports: [53]
    protocol: "udp"
    action: "permit"
    zone_from: "INSIDE"
    zone_to: "DMZ"
    business_justification: "Required for name resolution"
```

## Usage

```bash
# Generate firewall rules from policies
python fw_manager.py generate --format cisco
python fw_manager.py generate --format iptables

# Audit existing rules for issues
python fw_manager.py audit --input current_rules.txt

# Generate compliance report
python fw_manager.py report --output compliance_report.txt

# Show change log
python fw_manager.py changelog
```
